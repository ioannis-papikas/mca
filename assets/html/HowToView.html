Implementation How-To

This application implements the path-finding algorithm in 2 simple steps:

  1. Add or choose from a template the map to search into.
  2. Set a starting point and a goal point and hit 'Search'.

The algorithm performs a map pre-process before the first search for each map in order to create the search graph. This could take a while (5-10 seconds), so be patient. After the pre-process is done, each search does not take more than 2-3 seconds (it dependes on the map) to be completed.

Map Format

If you wish to create your own map and test the algorithm to custom paths, you have to format the map in a specific, but simple, structure so that it will be recognized properly.

In the first two lines declare the size of the map like this:

height 512
width 512

After the declaration lines, you present the map. The map must be represented in plain text. You can use any character you like, the map recognazies a valid path point as the symbol '.'. You can use any character you like for the walls. Example:

@@@@@@@@@@@@@@@@@@@@
@.........@.........
@...................
@.........@.........
@...@@@@@@@@@@@@....
@...@.....@....@....
@...@.....@....@@@@.
@.........@.........

Please limit the map size up to 512x512, for better display. The algorithm is tested for up to 2048x2048. Thank you.
